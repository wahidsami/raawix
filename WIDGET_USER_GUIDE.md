## Getting Started

### How to Access the Widget

1. **Look for the green button** in the bottom corner of the webpage
   - **English (LTR)**: Bottom-left corner
   - **Arabic (RTL)**: Bottom-right corner
2. **Click the button** to open the widget panel
3. **Or use keyboard**: Tab to the button and press Enter/Space

### Language Selection

- **Switch Language**: Click the language button (EN/AR) in the top bar
- **Arabic Mode**: Entire widget becomes RTL, voice speaks Arabic
- **English Mode**: Entire widget becomes LTR, voice speaks English

## Blind Users

### Scenario 1: First-Time User - Reading a Government Form

**Goal**: Fill out a government application form independently

**Steps**:

1. **Activate Voice Mode (Hands-Free)**:
   - Open widget → Tools tab
   - Select "Hands Free (Wake Phrase)" from Voice Mode dropdown
   - Grant microphone permission when prompted
   - Widget enters wake-only mode

2. **Wake the Widget**:
   - Say: **"هلا راوي"** (Arabic) or **"hi raawi"** (English)
   - Widget responds: "I'm here. Say: read page, summary, start form assistant."

3. **Enable Blind Preset**:
   - Widget asks: "Do you want me to enable Blind preset and start reading?"
   - Say: **"yes"** or **"نعم"**
   - Widget automatically:
     - Enables focus highlight
     - Enables reading mode
     - Increases text size
     - Starts reading the page

4. **Navigate the Form**:
   - Say: **"start form assistant"** or **"ابدأ مساعد النموذج"**
   - Widget guides you field by field:
     - "Field: Full Name. Required. Please enter the value."
     - You speak your name
     - Widget confirms: "Value entered: [your name]"
   - Say: **"next field"** or **"التالي"** to move forward
   - Say: **"repeat"** or **"كرر"** to hear the field again

5. **Handle Sensitive Fields**:
   - For National ID/Iqama, mobile, email:
     - Widget asks for double confirmation
     - You enter the value twice
     - Widget confirms both entries match

6. **File Uploads**:
   - Widget says: "This field requires a file upload. Now choose the file from your device."
   - Widget opens file picker
   - You select the file
   - Widget confirms: "File selected: [filename]. Continue?"

7. **Submit the Form**:
   - After all required fields are complete
   - Widget asks: "All required fields are complete. Ready to submit?"
   - Say: **"confirm submit"** or **"تأكيد الإرسال"**
   - Widget requires explicit confirmation before submitting

### Scenario 2: Reading a News Article

**Goal**: Read a long article independently

**Steps**:

1. **Wake Widget**: Say **"هلا راوي"** or **"hi raawi"**

2. **Start Reading**:
   - Say: **"read page"** or **"اقرأ الصفحة"**
   - Widget reads the entire page content

3. **Control Reading**:
   - Say: **"pause"** to pause
   - Say: **"resume"** to continue
   - Say: **"next"** to skip to next section
   - Say: **"repeat"** to hear current section again
   - Say: **"stop"** to stop reading

4. **Get Summary**:
   - Say: **"summary"** for a brief overview
   - Say: **"detailed summary"** for more details

5. **Navigate by Landmarks**:
   - Say: **"read landmarks"** to hear page structure
   - Say: **"go to section [heading name]"** to jump to a section

### Scenario 3: Understanding Page Actions

**Goal**: Know what actions are available on a page

**Steps**:

1. **Wake Widget**: Say **"هلا راوي"** or **"hi raawi"**

2. **List Actions**:
   - Say: **"what can I do here"** or **"ما الذي يمكنني فعله هنا"**
   - Widget lists top 5 actions with context:
     - "Action 1: Submit application about Contact Form."
     - "Action 2: View details about Service Card."

3. **Navigate Actions**:
   - Say: **"next action"** to move to next action
   - Say: **"previous action"** to go back
   - Say: **"activate action"** to click the current action

4. **Get Context**:
   - After hearing an action, say: **"why"** or **"لماذا"**
   - Widget explains why this action is relevant

### Scenario 4: Understanding Images

**Goal**: Know what images are on the page

**Steps**:

1. **Wake Widget**: Say **"هلا راوي"** or **"hi raawi"**

2. **Describe Image**:
   - Say: **"describe image"** or **"وصف الصورة"**
   - Widget describes the most relevant image (focused or topmost visible)
   - Uses assistive map descriptions if available from scan

3. **Describe Focused Element**:
   - Focus on any element (Tab navigation)
   - Say: **"describe focused element"** or **"وصف العنصر المحدد"**
   - Widget reads: element type, accessible name, role, state

## Low Vision Users

### Scenario 1: Reading Small Text

**Goal**: Make text larger and easier to read

**Steps**:

1. **Open Widget**: Click the green button

2. **Apply Low Vision Preset**:
   - Go to Assist tab
   - Click "Low Vision" preset button
   - Widget automatically:
     - Increases text size to 150%
     - Increases line spacing to 1.5x
     - Enables high contrast mode
     - Enables focus highlight

3. **Fine-Tune Settings**:
   - **Text Size**: Use +/- buttons to adjust (80% to 200%)
   - **Line Spacing**: Use +/- buttons to adjust spacing
   - **High Contrast**: Toggle on/off as needed

4. **Use Reading Mask**:
   - Go to Reading tab
   - Enable "Reading Mask"
   - Page dims except for a horizontal window around cursor
   - Adjust window height: Small / Medium / Large

5. **Use Magnifier**:
   - Go to Vision tab
   - Enable "Magnifier"
   - Move mouse to see enlarged content
   - Adjust zoom level with slider (150% to 500%)

### Scenario 2: Reducing Visual Clutter

**Goal**: Focus on content without distractions

**Steps**:

1. **Hide Images**:
   - Go to Reading tab
   - Enable "Hide Images"
   - Images replaced with placeholders (keeps layout)

2. **Stop Animations**:
   - Go to Reading tab or Vision tab
   - Enable "Stop Animations"
   - All CSS animations and transitions disabled

3. **Reduce Motion**:
   - Go to Vision tab
   - Enable "Reduce Motion"
   - Simulates prefers-reduced-motion (smooth scrolling disabled)

4. **Use Reading Guide**:
   - Go to Reading tab
   - Enable "Reading Guide"
   - Horizontal highlight bar follows mouse/focus
   - Helps track reading position

### Scenario 3: High Contrast Mode

**Goal**: Better visibility with high contrast

**Steps**:

1. **Enable High Contrast**:
   - Go to Vision tab
   - Enable "High Contrast Mode"
   - Colors invert for better contrast

2. **Combine with Other Features**:
   - Use with larger text
   - Use with reading mask for focused reading
   - Use with big cursor for easier mouse tracking

## Dyslexia Users

### Scenario 1: Reading with Dyslexia-Friendly Settings

**Goal**: Make text easier to read and process

**Steps**:

1. **Apply Dyslexia Preset**:
   - Open widget → Assist tab
   - Click "Dyslexia" preset button
   - Widget automatically:
     - Increases text size to 120%
     - Increases line spacing to 1.8x
     - Enables focus highlight
     - Enables reading mode

2. **Adjust Text Spacing**:
   - Increase line spacing to 2.0x for more breathing room
   - Adjust text size to comfortable level

3. **Use Reading Guide**:
   - Go to Reading tab
   - Enable "Reading Guide"
   - Highlight bar helps track reading line

4. **Use Reading Mask**:
   - Enable "Reading Mask"
   - Dims surrounding content
   - Focuses on current reading area
   - Adjust window height to comfortable size

5. **Voice Reading**:
   - Use voice mode to have content read aloud
   - Helps with comprehension
   - Can pause/resume as needed

### Scenario 2: Processing Long Documents

**Goal**: Break down long content into manageable chunks

**Steps**:

1. **Get Summary First**:
   - Wake widget: **"هلا راوي"** or **"hi raawi"**
   - Say: **"summary"** or **"ملخص"**
   - Understand overall content before reading details

2. **Read Section by Section**:
   - Say: **"read landmarks"** to hear structure
   - Navigate to specific sections
   - Read one section at a time

3. **Use Reading Mask**:
   - Focus on one paragraph at a time
   - Window height set to "Small" for single lines
   - Move through content gradually

## Motor Disabilities

### Scenario 1: Limited Mouse Control

**Goal**: Navigate and interact using keyboard only

**Steps**:

1. **Keyboard Navigation**:
   - Tab through all interactive elements
   - Widget is fully keyboard accessible
   - All buttons have focus indicators

2. **Voice Commands**:
   - Enable "Hands Free" mode
   - Use wake phrase: **"هلا راوي"** or **"hi raawi"**
   - Control everything by voice:
     - "read page"
     - "next action"
     - "activate action"
     - "start form assistant"

3. **Big Cursor**:
   - Go to Vision tab
   - Select "Big dark cursor" or "Big light cursor"
   - Easier to see and track cursor position

4. **Form Filling**:
   - Use Form Assistant with voice
   - No need to click or type
   - Widget fills fields based on voice input

### Scenario 2: Tremor or Shaky Hands

**Goal**: Reduce need for precise clicking

**Steps**:

1. **Large Click Targets**:
   - Widget buttons are large and easy to click
   - Focus indicators are prominent

2. **Voice Control**:
   - Use hands-free mode
   - Avoid clicking entirely
   - All actions via voice commands

3. **Keyboard Shortcuts**:
   - Tab navigation
   - Enter/Space to activate
   - Arrow keys for dropdowns

4. **Stop Animations**:
   - Prevents accidental clicks from animations
   - More stable interface

## Cognitive Disabilities

### Scenario 1: Understanding Complex Forms

**Goal**: Get step-by-step guidance

**Steps**:

1. **Use Form Assistant**:
   - Wake widget: **"هلا راوي"** or **"hi raawi"**
   - Say: **"start form assistant"** or **"ابدأ مساعد النموذج"**
   - Widget guides you through each field
   - Explains what each field needs

2. **Get Context**:
   - Say: **"what can I do here"** to understand available actions
   - Widget explains actions with context

3. **Review Before Submit**:
   - Say: **"review"** to hear all entered values
   - Confirm everything is correct
   - Widget requires explicit confirmation before submit

### Scenario 2: Reducing Cognitive Load

**Goal**: Simplify the interface

**Steps**:

1. **Hide Distractions**:
   - Hide images
   - Stop animations
   - Reduce motion

2. **Focus on One Thing**:
   - Use reading mask
   - Focus on one section at a time
   - Use reading guide to track position

3. **Voice Reading**:
   - Have content read aloud
   - Can pause to process information
   - Repeat sections as needed

## Quick Reference

### Voice Commands (English)

**Navigation**:
- "read page" - Read entire page
- "summary" - Brief summary
- "read landmarks" - List page structure
- "read actions" - List available actions
- "what can I do here" - List actions with context

**Form Assistant**:
- "start form assistant" - Begin form filling
- "next field" - Move to next field
- "previous field" - Go to previous field
- "repeat" - Repeat current field
- "skip" - Skip current field
- "review" - Review all entered values
- "confirm submit" - Submit form

**Reading Control**:
- "pause" - Pause reading
- "resume" - Continue reading
- "stop" - Stop reading
- "next" - Next section
- "repeat" - Repeat current section

**Wake Phrase**:
- "hi raawi" or "hey raawi" - Activate hands-free mode

### Voice Commands (Arabic)

**Navigation**:
- "اقرأ الصفحة" - Read entire page
- "ملخص" - Brief summary
- "اقرأ المعالم" - List page structure
- "اقرأ الإجراءات" - List available actions
- "ما الذي يمكنني فعله هنا" - List actions with context

**Form Assistant**:
- "ابدأ مساعد النموذج" - Begin form filling
- "التالي" - Move to next field
- "السابق" - Go to previous field
- "كرر" - Repeat current field
- "تخطي" - Skip current field
- "مراجعة" - Review all entered values
- "تأكيد الإرسال" - Submit form

**Reading Control**:
- "إيقاف مؤقت" - Pause reading
- "استئناف" - Continue reading

## Tips for Best Experience

1. **Grant Permissions**: Allow microphone access for voice commands
2. **Use Presets**: Start with a preset, then fine-tune
3. **Combine Features**: Use multiple features together (e.g., reading mask + voice reading)
4. **Practice Voice Commands**: Try commands in a quiet environment first
5. **Save Settings**: Settings persist in localStorage (if opted in)
6. **Switch Language**: Use language button to switch between Arabic and English

## Troubleshooting

**Voice not working?**
- Check microphone permission
- Ensure you're in "Hands Free" mode for wake phrase
- Try "Push to Talk" mode and click mic button

**Widget not appearing?**
- Check if page has loaded completely
- Look for green button in corner
- Try refreshing the page

**Settings not saving?**
- Settings only save if you opt in (when you enable voice mode)
- Check browser localStorage is enabled

**Wake phrase not working?**
- Make sure you selected "Hands Free (Wake Phrase)" mode
- Grant microphone permission
- Speak clearly: "هلا راوي" or "hi raawi"
